# 虎威传奇（Tiger Mir2 Red）

## 项目简介
虎威传奇（Tiger Mir2 Red）是基于经典 Mir2 服务器与客户端数据结构的复刻/私服项目，包含服务端配置、数据库、地图与脚本资源，以及若干自定义插件与管理工具。本仓库收纳了服务器相关的配置文件、地图数据、日志控制、以及运行脚本，方便搭建和维护私服。

## 主要目录说明
- `DBServer/`：数据库服务器相关文件与索引（保存游戏数据库文件与备份）。
- `LoginServer/`、`LoginGate/`、`RunGate/`：登录与网关服务配置，用于账号登录、排队及网关转发。
- `Mir200/`：客户端/服务端的资源文件，如地图、物品配置与脚本。
- `Envir/`：游戏环境配置目录，包含地图生成、怪物、任务、道具和管理员白名单等大量文本配置。
- `Log/`、`LogDataServer/`：日志文件目录（通常不纳入版本控制，见 `.gitignore`）。

## 编码与编辑器设置
本项目中文文件多以 GBK 编码保存。为了避免编码问题，已在工作区设置中将默认编码设为 GBK：
- `.vscode/settings.json` 中包含 `"files.encoding": "gbk"` 以及 `"files.autoGuessEncoding": true`。

如果你使用其他编辑器，请确保以 GBK 编码打开并保存这些文件。

## 快速上手
1. 根据你的环境（Windows / Linux）准备相应的服务器运行环境（老旧 Mir2 服务通常在 Windows 下运行）。
2. 检查 `Config.ini`、`LoginServer/LoginServer.ini`、`DBServer/DBServer.ini` 等配置文件，按需修改 IP、端口、数据库路径。
3. 启动数据库与服务端程序（注意：可执行文件、DLL 以及数据文件通常被忽略，不应提交到仓库）。

## 注意事项
- 仓库中包含大量配置与资源文件，请勿将可执行程序、数据库大文件或个人敏感信息提交到版本控制。`.gitignore` 已配置忽略常用二进制与日志目录。
- 如果需要导入/导出数据库备份，请先在本地测试环境验证备份完整性。
